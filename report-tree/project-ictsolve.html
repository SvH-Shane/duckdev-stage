    ictSolve is het eerste project waarbij ik volledig mee kon draaien. Dit gaf mij de kans om te ontdekken hoe het 1 en ander werkt. Bij dit project kreeg ik ook een stukje back-end mee.

    Om het project te starten, begonnen we als eerst met een meeting met de developers. We bespraken als eerst het ontwerp, die Roxanne aan de hand van de wensen van de klant had gemaakt.

    Er werd kort wat verteld over de keuzes van het ontwerp. En wat de klant graag zou willen zien.â€¨We bespraken de planning en daarna wie wat gaat doen. Dat verdelen ging vrij simpel, de website werd in blokken opgedeeld en daarvan kreeg iedereen een blok toegewezen.

    Ik kreeg de verantwoordelijkheid om alle de Wordpress in te richten en zo te maken dat deze gebruiksvriendelijk bewerkt kan worden door de klant




    Werkprocessen 1.1 1.3 1.5 2.1 2.3
    Custom fields
    Voor het inrichten van de Wordpress gebruikte we Custom Fields, die is een plug-in van Wordpress waarin je velden kan aanmaken. Daarbij een pagina zo kan inrichten dat de gebruiker het kan aanpassen.

    Ik moest voor ik aan het inrichten begon, eerst het ontwerp van de website opdelen. Ik moest kijken wat voor soort elementen er in zaten en hoe die element gebruikt zouden worden. Dus zou het element wat nu bovenaan staat ook een keer onderaan kunnen staan. En zou dat tekst veld wel eens een keertje herhaald kunnen worden?

    Toen ik dit uitgetekend had, en bepaald had of het logisch was. Ging in de velden in Wordpress aanmaken. Ik keek daarbij ook wat ik zelf prettig vond als ik hier als lake binnen zou komen, en of ik het direct snapte wat ik moest doen.

    Nadat ik eenmaal de basis had staan, liet ik dit nalopen door de andere. Die keken of wat ik had bedacht ook logisch voor hun was.

    Ik kreeg daarvan nog wat feedback wat direct erin had verwerkt, waardoor het door die aanpassingen ook nog een stuk logischer werd.


    Werkprocessen 2.1 2.3
    Custom fields omzetten
    Nadat dit alles wat goedgekeurd, moest dit hard in de code gezet worden. Zo kunnen wij straks veilig de custom fields verwijderen uit de Wordpress van de klant, zodat zij de opbouw niet kunnen aanpassen. Maar alleen de content.

    Hiervoor moest ik eerst de custom fields omzetten naar json en PHP file. Vervolgens moest ik wat bestanden aanmaken om en daarin konden de stukjes code.â€¨ Dit pushte ik vervolgens naar de test website die online staat.

    Uiteindelijk bleek dat de klant het soms niet helemaal prettig vond, en moest ik weer het lokaal aanpassen, om vervolgens de code weer naar de live versie te sturen.

    Na een paar keer wat aanpassingen te hebben gemaakt kwam hier dan ook samen met de klant geschikte velden uit, om in te bewerken.



    Werkprocessen 2.3 2.4
    API
    Die gegevens die werden ingevuld, moesten ook terug komen op de website. Daarvoor moest ik de API erbij pakken. En soort lange lijst die automatisch update als er nieuwe gegevens binnen zijn gekomen over de website.

    Elk veld had zijn eigen naam en zo stond dit ook in de API. Dit moest dan op een speciale manier terug komen in de code van website. Dit werkt dan mooi samen met het framework Nuxt.js wat Matise gebruikt om haar website te maken.â€¨â€¨In het begin een toch best lastige uitdaging. Maar met hulp kwam ik aardig op weg. En lukte het mij uiteindelijk om zelf ook gegevens terug te laten komen op de website.

    En gaf het toch best een kik, als het dan lukte.
