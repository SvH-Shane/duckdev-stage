    Voor dit project ben ik actief geweest in het pixel-perfect maken van de website. En het maken van een Singel-Page Dit houdt in dat dit een standaard pagina wordt, die steeds opnieuw aangemaakt en bewerkt kan worden.

    Werkprocessen 1.1 1.4 2.4
    Pixel-perfect.
    Met het pixel-perfect maken van de website, loop ik met behulp van Zeplin de website af. Zeplin heeft de functie om het ontwerp over de website heen te leggen. Zo zie je of alles klopt met het ontwerp, het gaat daarbij dus op de pixel precies.

    Als ik een fout zie, kijk in Zeplin wat de juiste maten horen te zijn. Vervolgens bekijk ik hoe dit in de code is verwerkt en pas ik het aan.

    Je leert hierbij dan ook om nauwkeurig te werken en om vooral om de kwaliteit van Matise te waarborgen. In het begin vond ik het frustrerend hoe hier opgelet wordt. Omdat ik dacht dat dit eigenlijk onmogelijk was om voor elkaar te krijgen. Maar niks is minder waar, doordat Matise een duidelijke documentatie heeft over hoe ze werken, is het eenvoudig te gebruiken. Daarnaast wordt het design goed op maat gemaakt, met een duidelijk grid en is het vrij eenvoudig om fouten te corrigeren.



    Scholtens in het Kort

    "Het viel mij op dat de stoelen altijd stuk waren en dat niemand goed wist hoe je dat op moest lossen. Ik wel, en na verloop van tijd repareerde en verhandelde ik stoelen. Goeie stoelen! In een vliegtuig doe je ook geen concessies wanneer het gaat om kwaliteit en veiligheid."

    Werkprocessen 1.1 2.1 2.4
    Single-Page
    Het maken van de Single-page was een echte uitdaging, ik begon bij een blanco pagina. Die natuurlijk vorm moest krijgen. De hoofdelementen zoals header, footer en geanimeerde knoppen waren al beschikbaar.

    Ook weer met behulp van Zeplin moest ik kijken hoe ik de pagina moest opbouwen. Het was een vrij eenvoudige pagina, dit komt doordat Matise zijn eigen grid systeem heeft gebouwd. In Zeplin kijk ik bijvoorbeeld op hoeveel grid een blokje staat. Stel je voor een #blokje staat 2 grid van de kant en is 5 grid breed, moet ik de code als volgt schrijven:

    #blokje {
    width: grid(5);
    margin-left: grid(2);
    }

    En zo zijn er nog meer handige tooltjes die Matise heeft ontwikkeld om een website nog sneller te kunnen maken om meer winst te halen uit een website.

    Als ik dan de juiste formaten heb ingevuld, volgt de rest eigenlijk vanzelf.

    Omdat het een Single-Page is, hoeft er niet veel aan gedaan worden. Het bestond in feite uit een groot blok en een header. Maar de manier van coderen toch best nieuw is, was ik er veel langer mee bezig dan ik had verwacht.

    Bij Matise gebruiken ze namelijk ook Nuxt.Js en dat is een totaal andere manier van het schrijven code. Dat samen met de techniek van Matise kan dat toch nog voor aardig wat hoofdpijn zorgen. Nuxt.Js is een JavaScript framework waar ik nog nooit mee had gewerkt. Hopelijk heb ik aan het eind van deze stage alles goed onder controle.

    Werkprocessen 2.1 2.4
    Omzetten in WP
    Uiteindelijk moest de statische pagina omgezet worden naar een Wordpress pagina. Hierbij moest ik vanuit de API content inladen op de pagina om deze dynamisch te maken.

    Aangezien ze dit vaker doen, kon ik kijken bij andere projecten hoe dat werd aangeroepen. Zo kreeg ik het ook dan ook voor elkaar dat beheerder een totaal andere paginanaam kan aanmaken en ook de content kan wijzigen.

    Het kan ook voorkomen dat de header niet altijd zichtbaar moest zijn. Ik zou dit zelf met PHP hebben opgelost, maar omdat het een javascript framework is kon dit ook anders. Hierdoor leerde ik weer een nieuwe manier om een if/else statement te maken.

    Ook hiermee was ik een tijd bezig en ben ik met hulp van collega's tot het eindresultaat gekomen om een pagina te maken die de beheerder zelf kan toevoegen, verwijderen en bewerken. En was ik toch best trots om op deze manier een bijdrage te kunnen leveren aan het project.
